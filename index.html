1

Clone it

bash
git clone https://github.com/lemma-work/lemma-platform.git
cd lemma-platform
2

Run the full stack locally

One command brings up everything in containers (needs Docker or Podman — the installer can set up Podman for you).

bash
curl -fsSL https://raw.githubusercontent.com/lemma-work/lemma-platform/main/install.sh | bash

When it's up, open http://127-0-0-1.sslip.io:3711 and create your account. (Use that host, not localhost — sign-in cookies are scoped to it.)

3

Install the CLI and point it at your local stack

bash
uv tool install lemma-terminal
lemma servers select local
lemma auth login

Which server? lemma servers select local points the CLI at the local stack you just started — that's what you build against on your machine. (To work against the hosted cloud instead, you'd use lemma servers cloud --use.)

4

Create your first pod

A pod is one self-contained workspace — tables, agents, workflows, and apps for one job.

bash
lemma pod create my-team --with-starter
lemma chat "what can you do in this pod?"
5

Use it from your own Claude Code or Codex

Two things your coding agent can do with a pod — build it, and power it:

Build: let your agent author pods for you

bash
lemma skills install

Drops Lemma's pod-building skills into Claude Code / Codex / Cursor, then just ask it to build a pod. This is all you need to build — no daemon required. (If it ever errors on symlinks, it's safe to skip.)

Run (optional): power the pod's agents with your subscription

bash
nohup lemma daemon start > ~/lemma-daemon.log 2>&1 &

Only needed when you want the pod's own agents to run on your Claude Code / Codex subscription instead of a configured model key. It bridges your local login to the pod — watch ~/lemma-daemon.log for ready ack. (We use nohup on purpose — lemma daemon start --background is currently flaky.)

6

Deploy it to Lemma (hosted)

Want your pod reachable by teammates and surfaces without self-hosting? Push it to lemma.work. Export your local pod, point the CLI at the cloud, and import it there:

bash
lemma pod export ./my-team
lemma servers cloud --use
lemma auth login
lemma pod import ./my-team

Built an app for the pod? Deploy that too (a no-build HTML file or a Vite project dir):

bash
lemma apps deploy my-app ./index.html
Credits: your lemma.work account comes with free starter credits — enough to deploy your pod and run it live on Lemma's hosted models. Prefer your own subscription on the cloud too? Just start the daemon while pointed at the cloud server.
That's it. You have Lemma running locally, a working pod, your own agent wired in, and a path to the cloud when you're ready.

1

Sign up for Lemma

Go to lemma.work, click Get started, and complete your sign-up or sign-in. New accounts get free starter credits automatically.

2

On "Connect your AI", choose "Connect a local harness"

Right after sign-up, Lemma asks how it should run AI for you. Pick the first option, Connect a local harness — that's your own Claude Code / Codex / OpenCode, connected through the Lemma daemon. Leave this screen open; you'll come back to it in a moment.

3

Open your coding agent, then paste this prompt

Open Claude Code (the app you set up in step 0), then paste the prompt below into it. It connects Lemma to your machine for you — the agent will pause and ask you to approve commands and to sign in in the browser, so just follow its plain-language instructions. You're done when it says "ALL STEPS ARE COMPLETE."

  • uv not found after install → run uv tool update-shell, or just open a fresh terminal, and continue.

  • No harness shows up after Recheck → make sure the agent finished ("ALL STEPS ARE COMPLETE") and the connection is still running. Re-run the daemon line and check the log for ready ack:
            
    nohup lemma daemon start > ~/lemma-daemon.log 2>&1 &
tail -n 20 ~/lemma-daemon.log
    
          
    • 
          
  • Reply errors asking for paid credits → switch the model to default (step 7).
    • 
          
  • On Windows → the background-connection command is newer and less tested. If it doesn't stay up, tell your agent your OS and ask it to keep the daemon running, or ping us.
    • 
          
  • Stuck anywhere → paste the error back to your coding agent; it can usually fix it on the spot.
